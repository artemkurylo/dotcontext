# Plan: dotcontext CLI — Milestone 1

Implements the `init` and `check` commands. No AI, no network — pure Go.

Related: [RFC-001: dotcontext CLI](../rfcs/001-cli.md) | [ADR-001: Go + Cobra](../adrs/001-go-cobra.md)

## Project Structure

```
dotcontext/
  cmd/
    dotcontext/
      main.go                # Entry point, cobra root command
  internal/
    init/
      init.go                # init command logic
      init_test.go
    check/
      check.go               # check command logic
      check_test.go
    templates/
      context.md.tmpl        # CONTEXT.md starter template
      adr.md.tmpl            # ADR template
      rfc.md.tmpl            # RFC template
      runbook.md.tmpl        # Runbook template
  go.mod
  go.sum
  Makefile
  LICENSE
```

Templates are embedded into the binary using Go's `embed` package — no external files needed at runtime.

## Command: `dotcontext init`

**Input:** current working directory
**Output:** `.context/` directory tree with starter CONTEXT.md

Logic:
1. Check if `.context/` exists. If yes and `--force` not set, print warning and exit 1.
2. Create directories: `.context/`, `plans/`, `stories/`, `rfcs/`, `adrs/`, `runbooks/`, `guides/`
3. Render CONTEXT.md template and write to `.context/CONTEXT.md`
4. Print summary of created files

Flags:
- `--force` — overwrite existing `.context/` directory

## Command: `dotcontext check`

**Input:** current working directory
**Output:** validation report on stdout, exit code 0 or 1

Checks (in order):
1. `.context/` directory exists
2. `.context/CONTEXT.md` exists and is non-empty
3. CONTEXT.md contains required section headings (configurable later, hardcoded for now):
   - `## What is this?` (or `## What is`)
   - `## Architecture`
   - `## Tech stack` (or `## Tech Stack`)
   - `## Status`
4. Files in `.context/adrs/` match pattern `NNN-*.md` (three-digit prefix)
5. Files in `.context/rfcs/` match pattern `NNN-*.md` (three-digit prefix)
6. Internal markdown links (`[text](../path)`) within `.context/` resolve to existing files

Output format:
```
dotcontext check

  PASS  .context/ directory exists
  PASS  .context/CONTEXT.md exists and is non-empty
  PASS  CONTEXT.md has required sections
  FAIL  .context/adrs/my-decision.md does not follow naming convention (expected NNN-slug.md)
  PASS  All internal links resolve

1 check failed
```

## Test Strategy

**Unit tests for init:**
- Creates all expected directories
- Creates CONTEXT.md with expected content
- Refuses to overwrite without --force
- --force overwrites existing structure

**Unit tests for check:**
- Passes on a valid `.context/` directory
- Fails on missing `.context/`
- Fails on missing CONTEXT.md
- Fails on empty CONTEXT.md
- Fails on missing required sections
- Detects badly named ADR/RFC files
- Detects broken internal links

**Integration test:**
- Run `init` in a temp directory, then run `check` — should pass

All tests use `t.TempDir()` for isolation.
